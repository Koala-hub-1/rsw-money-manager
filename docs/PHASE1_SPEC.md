# Phase 1: 実運用MVP仕様書

## 前提
Phase 0で作成した全コード・全テーブルをそのまま使用し、機能を追加する。
Phase 0のコードを捨てる部分はゼロ。

## 追加機能一覧

### 1. 認証・権限
- Supabase Authによるメール+パスワード認証
- 4ロール: rsw / admin / supervisor / family
- RLS（Row Level Security）を全テーブルに適用
  - RSWは担当入居者のみ閲覧
  - 家族は自分の親族のみ閲覧
  - adminは全件閲覧
  - supervisorは担当入居者のみ閲覧
- 招待リンクで家族・後見監督人のアカウント作成
  - adminが招待メール送信 → リンクからパスワード設定 → ロール自動付与

### 2. 入居者管理の拡張
- 入居者の新規登録画面（admin only）
  - 名前、カナ、生年月日、性別、要介護度、健康メモ、入居日、家族有無
- 入居者の編集画面（admin only）
- 任意後見契約情報の登録
  - 公正証書番号、後見人RSW、後見監督人
  - 月額振替上限（○○円）
  - デビット1回上限（○○円）
  - システム決済1回上限（○○円）
- RSW担当割当の管理

### 3. カードチェックアウト/チェックイン
- 物理カードの貸出記録
  - RSWがアプリ上で「チェックアウト」ボタンを押す
  - 対象カード選択 + 目的入力
  - チェックアウト時刻を記録
- 返却記録
  - 「チェックイン」ボタンを押す
  - チェックイン時刻を記録
  - 持出時間を算出
- 未返却一覧
  - ダッシュボードに未返却カード一覧を表示
  - 持出経過時間をリアルタイム表示

### 4. 基本アラート（3種類）

| アラート | 条件 | severity | 通知先 |
|----------|------|----------|--------|
| カード未返却 | checked_in_at == null かつ 24時間経過 | critical | RSW, admin |
| 残高不足 | current_balance < 月額固定費合計 | critical | admin, supervisor/family |
| レシート未添付 | デビットカード取引後24時間でreceipt_id == null | info | RSW |

- アラート一覧画面（severity順）
- 解決済みマーク機能
- ダッシュボードにアラートバッジ表示

### 5. 月次レポートPDF
- 入居者ごとに毎月1日に自動生成（Vercel Cron）
- PDF内容:
  - 対象月、入居者名
  - 収入合計（振替入金）
  - 支出合計
  - カテゴリ別支出（テーブル + 円グラフ）
  - 残高推移（折れ線グラフ）
  - アラート件数
- Supabase Storageに保存
- 家族/後見監督人ポータルからPDF閲覧・ダウンロード

## 追加画面

### ログイン画面 `/login`
- メール + パスワード入力
- ログイン後、ロールに応じてリダイレクト
  - rsw → `/`（ダッシュボード）
  - admin → `/admin`
  - supervisor / family → `/portal`

### 入居者管理 `/admin/residents`
- 一覧（検索、ステータスフィルタ）
- 新規登録フォーム `/admin/residents/new`
- 編集フォーム `/admin/residents/[id]/edit`
- 後見契約情報 `/admin/residents/[id]/contract`

### カードチェックアウト `/checkout`
- 担当入居者のカード一覧
- チェックアウトボタン（目的入力モーダル）
- チェックアウト中のカード一覧（タイマー表示）
- チェックインボタン

### アラート一覧 `/alerts`
- severity順に表示
- 解決済みマークボタン
- フィルタ（種別、入居者）

### 月次レポート `/residents/[id]/reports`
- 月選択
- PDFプレビュー or ダウンロード

### 家族/後見監督人ポータル `/portal`
- ダッシュボード（残高、直近取引、アラート）
- 取引履歴（Phase 0の入居者詳細と同等）
- 月次レポート閲覧・ダウンロード

### 招待管理 `/admin/invitations`
- 入居者を選択 → ロール選択（family / supervisor）→ メールアドレス入力 → 招待送信

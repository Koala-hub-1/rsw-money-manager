# RSW金銭管理システム — 全体要件定義書

## 0. プロジェクト概要

### 0.1 背景
アレソノワ株式会社（A.R.S.W.）が運営する「レジデンシャルソーシャルワーカーズ協会」において、RSW（社会福祉士）が住宅型有料老人ホーム入居者の金銭管理を行う。特に単身・家族なしの入居者では、RSWが身元保証人＝任意後見人として金銭管理の全権を担うため、不正防止と透明性確保のためのシステムが必須。

### 0.2 解決する課題
- RSWが入居者の金銭を直接扱うことによる横領・不正リスク
- 金銭管理の透明性が担保されない身元保証ビジネスの構造的問題
- 後見監督人・アレソノワ本部による監査の非効率性

### 0.3 設計思想
- **資金分離**: 入居者の本体口座にRSW・アレソノワはアクセスしない。生活費専用口座（住信SBI）を別途開設し、定額自動振替で資金を移動
- **決済ログ**: すべての支出がシステムを経由し改ざん不能な記録を残す
- **透明性**: 後見監督人（または家族）がリアルタイムで全取引を確認可能

## 1. 資金フロー

### 1.1 単身・家族なしの場合
```
本体口座（入居者名義・資産保全）
  ↓ 自動定額振替 ＆ 都度振替承認
生活費専用口座（住信SBI・入居者名義）
  ├── ① 口座振替（自動）→ 各サービス提供者（家賃・共益費・食費）
  ├── ② デビットカード（物理）→ 対面買い物（施設保管・チェックアウト制）
  └── ③ システム決済 → オンライン購入・Choice100（RSWは申請のみ）
```
- RSWは身元保証人＝任意後見人
- 定額振替の金額変更・都度振替はアレソノワ本部 + 後見監督人の承認が必要
- システムの役割: 入金監視・都度振替の承認ワークフロー

### 1.2 家族ありの場合
```
本体口座（入居者名義）← RSW・アレソノワともにアクセス不可
  ↓ 家族が毎月振込
生活費専用口座（住信SBI・入居者名義）
  ├── ① 口座振替（自動）
  ├── ② デビットカード（物理）
  └── ③ システム決済
```
- 家族が振込管理
- システムの役割: 入金確認の監視・入金遅延時アラート

### 1.3 3つの支出チャネル

| チャネル | 決済手段 | RSWの関与 | システム制御 |
|----------|----------|-----------|-------------|
| ① 口座振替 | 自動引落 | なし | CSV取込で記録 |
| ② デビットカード（物理） | 施設保管→チェックアウト制 | カード持出・返却 | 貸出ログ・利用通知 |
| ③ システム決済 | バーチャルカード（システム経由） | 申請のみ | 承認フロー・カード番号隠蔽 |

## 2. ユーザーロールと権限

| ロール | 略称 | 説明 |
|--------|------|------|
| RSW | rsw | 担当入居者の日常金銭管理 |
| アレソノワ本部 | admin | 全拠点横断管理 |
| 後見監督人 | supervisor | 入居者ごとの監査 |

## 3. フェーズ計画

### Phase 0: プロトタイプ（6日間）
- freee CSV取込（銀行口座・デビットカード明細）
- 入居者2名のモックデータ
- 口座残高・カード残高の一覧表示
- 取引明細 + カテゴリ自動推測
- スマホからレシート撮影・明細紐づけ
- 支出内訳グラフ（カテゴリ別）

### Phase 1: 実運用MVP（+4〜6週間）
Phase 0に以下を追加:
- 認証（Supabase Auth）+ ロール分離（RSW / admin / supervisor / family）
- 招待リンクで家族・後見監督人のアカウント作成
- 入居者の新規登録・編集画面（admin）
- 任意後見契約情報の登録（上限金額の設定）
- 物理カードの貸出・返却記録
- 基本アラート（カード未返却24h / 残高不足 / レシート未添付24h）
- 月次レポートPDF自動生成（カテゴリ別支出・残高推移・アラート件数）

### Phase 2+: フル機能（順次追加・検証しながら）
- 購入申請 → 承認ワークフロー
- 都度振替の承認ワークフロー
- LINE通知（リアルタイム）
- 領収書OCR（Google Cloud Vision）
- 高度な異常検知（同一店舗頻度 / 月間超過等）
- RSW別パフォーマンスダッシュボード
- Expoモバイルアプリ（RSW専用）
- スマートカードキャビネット（ESP32 BLE連携）
- Choice100サービスカタログ
- 監査レポート自動生成

## 6. セキュリティ要件

- Supabase Auth（メール + パスワード）
- RLSで全テーブルに行レベルアクセス制御（Phase 1）
- バーチャルカード番号はpgcryptoで暗号化（Phase 2）
- audit_logsテーブルはINSERT ONLY（UPDATE/DELETE禁止）
- セッションタイムアウト: RSWアプリ30分、Web 60分
- 領収書画像はSupabase Storage（private bucket）
- 全APIリクエストにユーザーIDをログ
- ログ保持期間: 7年
